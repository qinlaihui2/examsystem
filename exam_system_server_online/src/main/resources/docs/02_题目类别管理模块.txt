1.获取轮播图详情接口
  接口分析：
      参数：
          无
      响应：
          {
            "code": 200,
            "message": "操作成功",
            "data": [
              {
                "id": 1,
                "name": "Java基础",
                "parentId": 0,
                "sort": 1,
                "children": null,
                "count": 50  # 题目数量
              }
            ]
          }
  业务分析：
     查询所有的非逻辑删出的分类信息集合（sort字段正序排序）
     查询分类对应的题目的数量
     是一个多表查询（题目分类表+题目表）
     方案1：多表查询（分类信息+连接+题目分组查询）
     最简单：写好多表sql就行了，问题就是嵌套子查询效率较低

     方案2：分步查询（单表查询分类信息-》每一个分类ID-》题目表题目的数量）
        mybatisplus分布查询触发
        循环分类每条数据查询下的题目数量
        1+n问题
     方案3：分类表集合+题目中分类id和题目数量java代码中进行数据逻辑处理
  实现过程：
     1.查询所有的分类信息集合（单表操作）-》list
     2.QuestionMapper定义查询方法，category_id进行分组，并且统计每个分类下的题目数
     list<<Map>
        map->key->category||key->counut
     2.题目查询的分类题目数量赋值给分类集合
     4.返回查询集合
  注意事项：
     注意排序问题
     自定义mapper方法，sql语句中需要中间添加逻辑删除语句
